## Evidence Relevance Assessment (CRITICAL)

For EACH evidence item, assess:
1. Does this evidence DIRECTLY address the question being asked?
2. Could this measurement be identical regardless of the correct answer?
3. What specific aspect of the question does this evidence address (or fail to address)?

Mark evidence as LOW RELEVANCE (score < 0.5) if:
- It measures a property not related to the question
- It could be the same whether the answer is A or B
- It addresses a different aspect of the audio than what's being asked

### 1. Inter-tool Contradictions (between different evidence sources)
Look for evidence items that make conflicting claims about the same thing:
- Different counts of the same phenomenon
- Different identifications of the same sound source
- Conflicting environment classifications

### 2. Intra-tool Contradictions (within a single evidence item)
Check if a tool's CONCLUSION contradicts its own listed EVIDENCE:
- A classification that conflicts with the supporting measurements
- A summary that contradicts the detailed findings
- A confidence level inconsistent with the reported uncertainty

When you find intra-tool contradictions, mark that evidence as potentially_hallucinated.

### 3. Reliability Hierarchy Violations
When evidence conflicts, higher reliability sources should be trusted:
- REPRODUCIBLE tools (beat detection, energy analysis) give consistent measurements but require relevance assessment
- PROBABILISTIC tools (ASR, diarization) are usually reliable but can err
- HEURISTIC tools (classification, environment detection) provide useful but uncertain signals
- SPEECH_LLM claims are speculative and need corroboration — but should NOT be dismissed outright. When uncorroborated, propose segment-specific verification in hypotheses_to_test rather than marking as hallucinated.

IMPORTANT: When two HEURISTIC tools contradict each other, BOTH should be marked as potentially_hallucinated since neither has higher authority.

IMPORTANT: Contradictions from reproducible tools should be MARKED but not used to DISMISS.
- Reproducible tools may analyze different segments or have timing offsets
- Mark the contradiction as "evident" but keep both pieces of evidence
- Let the argumentation agent weigh the contradiction in final reasoning
- Example: If energy dynamics shows "no sudden changes" but speech LLM reports something loud at 3s":
  - Mark as contradiction type="reliability_violation", severity="medium"
  - Do NOT mark speech LLM claim as "hallucinated" - the timing could differ
  - The final argumentation will consider both perspectives

### 4. Logical Reasoning Awareness

When analyzing evidence, be aware of these reasoning pitfalls:

**Absence Claims**: Evidence stating "X not detected" has lower reliability than evidence stating "X detected".
A tool failing to detect something does NOT confirm its absence - tools have detection thresholds and limitations.

**Single-Source Speculative Claims**: SPEECH_LLM evidence with specific scenario descriptions (settings, contexts,
intentions) should be marked as speculative and flagged for segment-specific verification, not dismissed as potentially_hallucinated. Only mark as potentially_hallucinated if actively contradicted by reproducible tool evidence analyzing the same segment.

**Diarization Speaker Counts**: If diarization reports a speaker count but multiple independent observations
describe interactions between more participants, the diarization count may be unreliable.

**Transcription Disagreements**: When different tools/sources report different transcriptions for the same segment:
- Flag as contradiction ONLY if the segment timings significantly overlap (within 1-2 seconds)
- Note the EXACT words that differ, especially negation words ("don't" vs "do", "not" vs "")
- Mark severity as "high" when a single word changes the meaning completely
- If timings don't overlap, the different transcriptions may be for different segments - not a contradiction
